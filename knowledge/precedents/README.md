# Internal Precedents

Anonymized distillations of past matters. Use these as a starting point when a similar fact pattern arrives.

## How to Add a Precedent

After closing a matter, if the analysis would be useful for future reference:

1. Create a new file in this folder: `<topic-slug>.md` (e.g., `athletes-non-resident.md`)
2. Use the structure below
3. **Strip all identifying information**: client names, specific dates that would identify, specific amounts unless illustrative
4. Focus on the legal reasoning and methodology, not the specific facts

## Precedent Template

```markdown
# [Topic]

## Fact Pattern

[Generic description of the situation — non-resident athlete, seconded employee, etc.]

## Key Question

[What was the legal question?]

## Analysis

### Applicable Statutes
- Art. X NK AR
- Art. Y of Law Z

### Reasoning
[Step-by-step legal analysis]

### Conclusion
[The answer]

## Conditions

[What conditions must be met for the conclusion to apply?]

## Pitfalls and Edge Cases

- [Common misinterpretations]
- [What changes if facts differ in X way?]

## Tags

`tax-pit`, `non-resident`, `sport`, `art-102-1-15`
```
